Does it have to be "tidyquant"?  Because that kind of
computation can be done easily (one function call) with
package "zoo":


    ## fetch data and make it a zoo series
    library("tidyquant")
    library("zoo")
    bhp <- as.data.frame(tq_get("bhp"))
    z <- zoo(bhp[, 3:8], bhp[, 2])


    ## compute 250-day maximum for all numeric columns:
    ## close, volume, etc.
    m <- rollapply(z, width = 250, FUN = max,
                   by.column = TRUE, align = "right")


    ## visually inspect result
    plot( z[, "close"], ylab = "Close")
    lines(m[, "close"], col = grey(.5))

Anatoly,

The default solver for many common targets was switched to CVXR because
CVXR tends to be faster and have more capability for many common
patterns than the ROI solver which we used as the default for most of
the default optimization patterns for several years.

The first answer is that you can always specify the 

optimize.portfolio='ROI'

in your prior instructions and things should "just work" and go back to
the old default behavior.  You might choose to just start here and
specify ROI, as PortfolioAnalytics still supports ROI, and it should
work now for anything it previously worked for.

The more complicated answer is that CVXR rewrote their interfaces
between their 1.0x versions and the 1.8.x versions from early March of
2026 as they have rewritten the internals using R7 in 1.8.x

Overall, I am sure that this will improve CVXR.  Right now, there is
some friction which you have discovered. In the 1.8.x announcement of
the new R7 functionality, the CVXR team has created breaking changes in
their interfaces.  Their 1.8.x announcement   said that they would
maintain backwards compatibility for a time, but their very next point
release (intentionally or unintentionally) broke backwards
compatibility, as you have found.  CRAN's reverse dependency checks did
NOT find this, unfortunately.

I'm preparing a new version of PortfolioAnalytics which adapts to the
new CVXR interfaces.  I hope to push the new PortfolioAnalytics to CRAN
"soon".

Hopefully specifying ROI as your optimize_method solves your problem
while I work to get a new version of PortfolioAnalytics to CRAN that
adapts to the CVXR breaking changes.
